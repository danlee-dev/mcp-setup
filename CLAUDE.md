# Claude Code 사용 규칙

## 기본 규칙
- 이모지 사용 금지
- AI스러운 표현 지양
- 간결한 응답

## Git Commit 규칙
- Conventional Commits 형식 준수
- 접두사: feat, fix, docs, style, refactor, test, chore
- 예시: `feat: add github mcp server support`
- Claude 작성 표기 금지

## 커밋 메시지 형식
```
<type>: <description>

[optional body]
```

## 허용된 접두사
- feat: 새로운 기능
- fix: 버그 수정
- docs: 문서 변경
- style: 코드 스타일 변경
- refactor: 코드 리팩토링
- test: 테스트 추가/수정
- chore: 빌드/설정 변경

## 응답 스타일
- 간결하고 직접적
- 불필요한 설명 최소화
- 이모지나 AI스러운 표현 금지
